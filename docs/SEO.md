# SEO file generation

How `public/sitemap.xml` and `public/robots.txt` are produced and kept in sync with the app.

## Overview

Both files are **generated by a Node script**, not maintained by hand. That keeps every public URL aligned with the same inventory used for verification.

| Output | Path | Generator |
|--------|------|-----------|
| XML sitemap | `public/sitemap.xml` | `scripts/generate-seo.ts` |
| robots.txt | `public/robots.txt` | `scripts/generate-seo.ts` |

Files in `public/` are served from the site root at `https://www.futureearningsexpert.com/sitemap.xml` and `/robots.txt`.

## URL inventory (`buildPublicUrlInventory`)

The script imports `buildPublicUrlInventory` from `src/lib/seo/publicUrlInventory.ts`. That function assembles all paths that should appear in search, then maps them to absolute URLs with canonical host `https://www.futureearningsexpert.com`.

The inventory merges:

- **Static app routes** — `APP_STATIC_PATHS` (home, services hub, pillar page, hubs, resources, etc.)
- **Dynamic routes** — slugs from `services`, `damages-contexts`, `earnings-damages-types`, `case-types`, and `guides` content modules

Excluded from the sitemap (but still valid pages): `/contact`, `/thank-you`, `/privacy`, `/terms`.

The combined list is deduplicated, sorted, and exposed as `allUrls` (absolute) and `allPaths` (pathname only).

## Sitemap generation

For each URL in `inventory.allUrls`, the script emits a standard `<url>` entry with:

- `<loc>` — absolute URL
- `<lastmod>` — calendar date when generation runs (UTC)
- `<changefreq>` — heuristic by path
- `<priority>` — heuristic by path (home 1.0, key hubs higher, glossary lower)

## robots.txt generation

`robots.txt` is a fixed template in the same script. It:

- Allows all crawlers on `/`
- Disallows `/thank-you`, `/privacy`, `/terms`, `/contact`
- Declares `Sitemap: https://www.futureearningsexpert.com/sitemap.xml`

## Commands

```bash
npm run seo:generate   # Regenerate sitemap.xml + robots.txt
npm run seo:verify     # Fail if sitemap is missing URLs vs inventory
npm run build          # Runs seo:generate automatically before next build
```

Run `seo:generate` after adding static routes to `APP_STATIC_PATHS` or before release when you want fresh `lastmod` dates.

## Adding new pages

1. Create the Next.js route.
2. If static, add the path to `APP_STATIC_PATHS` in `src/lib/seo/publicUrlInventory.ts`.
3. If dynamic, ensure the slug is exported from the relevant content module (already picked up automatically).
4. Run `npm run seo:generate` and commit `public/sitemap.xml` and `public/robots.txt`.

## Note on Next.js

This project uses **static files in `public/`** rather than `app/sitemap.ts` / `app/robots.ts`, so the committed generated files are the source of truth for crawlers.
